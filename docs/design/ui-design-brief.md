# Lumen 外观设计简报（UI Design Brief）

> 用途：给设计工具 / 设计师做**视觉外观设计**。本文只讲「有哪些页面/面板、每个上面有什么、有哪些状态、什么气质」，不涉及实现代码。
> 依据：`client-design.md`（客户端权威设计）、`web-design.md`（官网）、`protocol-design.md`（能力契约）、`00-overview.md`（路线图）。
> 平台：**Windows 桌面客户端**（Wails v2 + WebView2，渲染能力 = 现代 Chromium，可用一切现代 CSS/动效）为主；**官网**为纯静态响应式 SPA 为次。
> 版本标记：`[v0]` 最小可用（先设计）、`[v1]` 完整体验、`[v2]` 加密（暂不设计）。

---

## 0. 一句话产品定位
类 Discord 的**轻量语音聊天工具**，面向小圈子「开黑」（游戏语音）。核心体验：**进频道、说话、看谁在说、发字**——低打扰、常驻托盘、游戏全屏下也能用。

---

## 1. 设计语言（Design Language）

| 维度 | 取向 |
|------|------|
| 主题 | **深色为默认且唯一**（v0/v1）；预留亮色切换位（v1 可只做深色） |
| 气质 | 紧凑、信息密度偏高（类 Discord）；游戏场景友好；实时感强 |
| 重点视觉 | **说话高亮**（头像描边强调色 + 轻微发光）、静音/闭麦状态、连接状态 |
| 语言 | **中文优先**，无衬线字体；时间/数字可用等宽 |
| 打扰度 | 尽量非阻断（toast）；仅鉴权失效等 CRITICAL 用模态阻断 |
| 无障碍 | 深色下保证对比度、清晰焦点态、图标配文字/tooltip |

**建议主题 Token（给设计系统落地）**
- 背景层级：`bg-app` / `bg-panel` / `bg-elevated`（三级深灰）
- 前景：`fg-primary` / `fg-muted`
- 强调：`accent`（主色）、`accent-glow`（说话发光）
- 语义：`success`、`warn`、`danger`（离开/删除/被踢）
- 语音态：`speaking`（=accent）、`muted`、`deafened`
- 形状/动效：统一圆角、面板阴影、说话「呼吸发光」动效、连接「脉冲」动效

## 2. 桌面客户端 · 全局框架

```
┌───────────────────────────────────────────────────────────────┐
│  顶部连接状态条  connecting / reauth / reconnecting / closed     │  ← 仅异常/过渡时出现
├──────────┬──────────────────────────────────┬─────────────────┤
│ Sidebar  │            主内容区               │  成员/语音侧栏    │
│ 频道侧栏  │  [文字频道] 消息列表 + 输入框      │  语音在线成员     │
│ #大厅    │  或                               │  + 成员列表       │
│ #公告    │  [语音频道] 成员网格              │                 │
│ 🔊开黑1  │                                   │                 │
│ 🔊开黑2  │                                   │                 │
│ ─────    │                                   │                 │
│ [我 ⚙]   │                                   │                 │
│ ControlBar：🎤自静音 🔈扬声静音 ⏻离开 ⚙设置                     │
└───────────────────────────────────────────────────────────────┘
```

- **三栏**：左频道侧栏 / 中主内容（文字或语音）/ 右成员&语音侧栏。
- **顶部连接条**：平时隐藏；`connecting`（连接中）/`reauth`（刷新令牌中）/`reconnecting`（重连中，带退避倒计时）/`closed`（断开）四态。
- **底部控制条**：常驻，跨全局。
- 无原生浏览器边框；窗口设最小尺寸；可最小化到托盘。

## 3. 页面 / 面板清单（逐个）

### 3.1 登录页 `LoginView` `[v0]`
| 元素 | 说明 |
|------|------|
| 品牌区 | Logo + 「Lumen」名 + 一句 slogan（开黑语音） |
| 登录按钮 | 点击→拉起系统浏览器完成登录（handoff）；按钮进「登录中」态 |
| 状态/错误 | 登录中（提示"已在浏览器打开，请完成登录"，可取消）、失败 toast |
| 页脚 | 版本号、隐私/条款链接（指向官网） |

**状态**：未登录默认 / 登录中（等待浏览器回调，可取消）/ 失败（网络不可用、超时 3 分钟、校验失败）/ 成功→进入主界面。

### 3.2 主界面骨架 `MainLayout` `[v0]`
承载 3 栏 + 顶部连接条 + 底部控制条；登录后默认选中第一个文字频道（如「大厅」）。

### 3.3 频道侧栏 `Sidebar` / `ChannelList` `[v0]`列表 · `[v1]`增删改
| 元素 | 版本 | 说明 |
|------|------|------|
| 文字频道项 `# 名称` | v0 | 选中态高亮；点击切换主内容区 |
| 语音频道项 `🔊 名称` | v0 | 点击=加入该语音频道；显示已在此频道的成员数/小头像 |
| 频道分组/顺序 | v0 | 按 `position` 排列，文字/语音分组 |
| 「+ 新建频道」 | v1 | 仅 owner 可见 |
| 频道右键菜单（重命名/删除） | v1 | 仅 owner |
| 底部「自我卡片」 | v0 | 头像 + 昵称 + 连接/语音状态；齿轮 ⚙ 进设置 |

**空态**：无频道时提示（一般不会，服务端种子有「大厅/开黑1」）。

### 3.4 文字频道 `TextChannel` / `MessageList` `[v0]`
| 元素 | 说明 |
|------|------|
| 顶部标题 | 当前频道名 `# 大厅` |
| 消息列表 | 升序；**顶部「加载更多」**分页；消息行 = 头像 + 昵称 + 时间 + 正文 |
| 输入区 | 多行输入框 + 发送按钮（Enter 发送）；发送中/失败重试 |

**状态**：加载首页中 / 空频道（"还没有消息，来说点什么"）/ 加载更多中 / 到顶（无更多）/ 发送失败（行内重试）。头像昵称随资料更新实时刷新。

### 3.5 语音面板 `VoicePanel` / `MemberItem` `[v0]`基础 · `[v1]`逐人音量
| 元素 | 版本 | 说明 |
|------|------|------|
| 在线成员（网格/列表） | v0 | 头像 + 昵称 |
| **说话高亮** | v0 | 正在说话者头像描边强调色 + 发光 |
| 静音/闭麦图标 | v0 | 🔇 自静音、🎧✖ 闭麦（deafen） |
| 逐人音量滑块 | v1 | 每个远端成员一个音量条（0~400%），拖到 0 = 本地静音该人 |
| 语音重连提示 | v0 | 通话中断时面板顶部"正在重连语音…" |

### 3.6 成员侧栏（在线/全部成员）`[v0]`
频道成员列表：在线优先；头像 + 昵称 +（可选）状态。与语音面板同处右栏（语音频道时上半区为语音成员、下半区为全员）。

### 3.7 底部控制条 `ControlBar` `[v0]`
| 按钮 | 说明 |
|------|------|
| 🎤 自静音 | 切换自己麦克风；静音态明显变色 |
| 🔈 扬声静音（deafen） | 一键静音所有人（同时自静音） |
| ⏻ 离开语音 | 退出当前语音频道 |
| ⚙ 设置 | 打开设置弹窗 |
| 自身说话指示 | 自己在说话时的电平/高亮 |
| 模式标识 | 当前 PTT / VAD / 常开（v1） |

### 3.8 设置弹窗 `SettingsModal` `[v1]`（分 Tab）
1. **音频 `AudioSettings`**：输入设备选择、输出设备选择、**麦克风测试**（实时电平条）、降噪开关（RNNoise）。
2. **语音模式 `VoiceMode`**：PTT / VAD / 常开 三选一；**PTT 键绑定**（点击捕获组合键，冲突/占用报错回滚）；**全屏游戏模式**开关（低层键盘钩子）；VAD 阈值（进入/退出/hangover 三滑块，带实时电平参考）。
3. **应用 `AppSettings`**：主题（深色，预留亮色）、开机自启、全屏游戏模式、**检查更新**（当前版本 / 有新版本时显示版本号+更新说明+下载进度）。

### 3.9 系统集成视觉（非独立页面，但需出图）`[v1]`
- **系统托盘**：图标 + 右键菜单（显示主窗口 / 麦克风静音「勾选态」/ 退出）。
- **最小化到托盘**：关闭=隐藏到托盘（非退出）。
- **更新可用**：非阻断横幅/toast「有新版本」→ 下载进度条 → 安装重启提示。
- **托盘/任务栏麦克风状态**：与应用内自静音同步。

### 3.10 全局组件
| 组件 | 版本 | 说明 |
|------|------|------|
| Toast（error/warn/info） | v0 | 右下角非阻断；错误文案用服务端中文 |
| 模态阻断 | v0 | 仅 CRITICAL（登录过期/被踢）→ 回登录 |
| 连接状态条 | v0 | 见 §2 四态 |
| 通用加载/空/错误态 | v0 | 统一骨架屏/占位 |

## 4. 需要专门出视觉的「状态」清单
- 说话高亮（描边+发光，动效）
- 自静音 / 扬声静音 / 被静音（本地静音某人）三种图标语义
- PTT 按下态（自身门控开）
- 连接四态（connecting / reauth / reconnecting+倒计时 / closed）
- 语音重连中
- 逐人音量滑块（含 0=本地静音）
- 麦克风测试电平条 + VAD 阈值可视化
- 消息发送中 / 发送失败重试
- 消息分页「加载更多 / 到顶」
- 空态：空频道、无成员
- 异常回登录：登录过期、被踢（冷却提示）
- owner 专属 UI（新建/删除频道、踢人入口）——非 owner 隐藏

## 5. 官网静态站（次要，同一视觉语言）`[v0]`

纯静态响应式 SPA（桌面 + 移动）。页面：

| 页面 | 路径 | 内容 |
|------|------|------|
| 首页 Home | `/` | 营销：开黑卖点、截图、下载 CTA |
| 下载 Download | `/download` | Windows 安装包下载 + 版本/校验信息 |
| 账户中心 Account | `/account` | 登录后看头像/昵称、退出（未登录跳登录） |
| 帮助 Help | `/help` | 使用说明/FAQ |
| 隐私 Privacy | `/privacy` | 隐私政策 |
| 条款 Terms | `/terms` | 服务条款 |
| 关于 About | `/about` | 项目介绍 |
| 404 NotFound | `*` | 找不到页面 |

- **账户中心登录**：点登录→整页跳转到后端 `/auth/login`（外部 IdP）→回跳显示资料。
- 视觉与客户端一致（深色主色可复用），但更「官网/营销」气质。

## 6. 版本切分（先设计什么）

**先做 `[v0]` 这批屏**（能跑通「登录→看频道→发字→进语音说话」）：
登录页、主框架（3 栏+连接条+控制条）、频道侧栏（列表）、文字频道+消息列表、语音面板（基础：头像/说话高亮/静音图标）、成员侧栏、控制条、toast/空态/连接态。

**再做 `[v1]`**：设置弹窗全部 Tab、逐人音量滑块、频道 CRUD/右键菜单、owner 专属入口、托盘/最小化/更新视觉、PTT/VAD 模式相关 UI。

`[v2]`（E2E 加密）：本期**不设计**。

---

## 7. 给设计工具的约束提示
- 深色默认；类 Discord 的三栏 + 底部控制条布局。
- 目标分辨率：桌面窗口（建议最小 ~1000×640，常见 1280×800+）；官网另做响应式。
- 渲染环境 = Chromium（WebView2），现代 CSS/圆角/阴影/动效/`backdrop-filter` 均可用。
- 图标建议统一一套线性图标；语音/静音/连接等状态图标语义要清晰可辨。
- 说话高亮、连接脉冲、更新进度这类**动效**请给出规格（时长/缓动）。
- 中文文案为主；预留多字号（信息密度）与 hover/tooltip。
